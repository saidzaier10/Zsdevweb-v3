# Plan d'Optimisation : Animation d'Arrière-plan

## Objectifs
1.  **Visibilité** : Rendre l'animation visible sur toutes les pages en supprimant les fonds opaques.
2.  **Esthétique** : Enrichir la palette de couleurs pour un effet plus vibrant.
3.  **Performance** : Réduire la consommation d'énergie (FPS limit, pause en arrière-plan).

## Changements Proposés

### 1. Visibilité & Couleurs
**Fichier :** `frontend/src/App.vue`
- Augmenter l'opacité du conteneur d'arrière-plan (`opacity-40` -> `opacity-70`).

**Fichier :** `frontend/src/views/Home.vue`
- Supprimer les classes `bg-white` et `dark:bg-dark-900` du conteneur principal pour le rendre transparent.

**Fichier :** `frontend/src/components/ui/InteractiveBackground.vue`
- **Palette de couleurs** : Ajouter des couleurs plus vibrantes (Rose, Cyan, Violet, Bleu électrique).
- **Taille** : Légère augmentation de la taille des particules (min 2px, max 4px).

### 2. Optimisation des Performances
**Fichier :** `frontend/src/components/ui/InteractiveBackground.vue`
- **FPS Limit** : Implémenter une boucle d'animation contrôlée à 30 FPS.
- **Page Visibility** : Arrêter l'animation (`cancelAnimationFrame`) quand `document.hidden` est vrai.
- **Particules** : Ajuster le nombre de particules (60 desktop / 30 mobile).

## Plan de Vérification

### Test Visuel
- Naviguer sur la page d'accueil.
- Vérifier que les particules colorées sont bien visibles derrière le contenu.
- Vérifier que l'animation est fluide mais pas trop rapide.

### Test Performance
- Ouvrir un autre onglet, revenir : vérifier que l'animation reprend sans bug.
- Vérifier dans le Task Manager du navigateur que la charge CPU est faible.
